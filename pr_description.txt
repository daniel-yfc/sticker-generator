🧹 [code health improvement] Refactor App State into Custom Hook

🎯 **What:** Extracted the complex state management and action handlers from `App.tsx` into a new custom hook called `useAppState`. Additionally, updated the `id` generation in `addToHistory` from using `Math.random()` to `crypto.randomUUID()` as per memory instructions for better cryptographic security.

💡 **Why:** `App.tsx` had grown overly complex, mingling state definitions, effect hooks, and UI layout code. By extracting the state logic into `hooks/useAppState.ts`, the main component is now significantly more readable, easier to test, and focused purely on layout and composition. This separation of concerns improves long-term maintainability.

✅ **Verification:**
- The React component `App.tsx` was refactored without any syntax errors.
- The project successfully builds (`npm run build`).
- The full test suite passes (`npm run test`), ensuring no existing functionalities or interactions were broken.
